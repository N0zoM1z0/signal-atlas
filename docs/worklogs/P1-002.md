# P1-002 Worklog: Phaser world scene

## Status

Complete.

## Goal

Replace the decorative React world rendering with a real Phaser-owned 48 × 30 pixel scene while preserving a semantic DOM mirror and a strict typed boundary to React.

## Intended files

- `packages/game-scene`: scene definition, typed bridge, integer-scaling geometry, Phaser lifecycle, rendering, camera, input, and unit tests.
- `apps/web/src/world-shell`: lazy canvas host, toolbar wiring, DOM mirror synchronization, reduced-motion propagation, and fixture scene model.
- `tests/e2e/world-shell.spec.ts`: canvas readiness, camera, DOM mirror, performance, and visual regression coverage.

## Boundaries

- Phaser owns canvas rendering, building hit zones, ambient layers, camera state, and short scene callouts.
- React owns accessible controls, selected view state, panels, and every authoritative domain mutation.
- The bridge carries typed view commands and scene interaction events only; it cannot import or call application services.
- Palette values are read from generated CSS custom properties. The scene does not embed an independent raw color palette.
- Agent markers are minimal camera targets in this task; character sprites, states, and full bidirectional selection belong to P1-003.

## Acceptance criteria

- The authored world renders as a crisp, integer-scaled Phaser canvas at target desktop sizes.
- Six fixture places, routes, sky, clouds, lit windows, weather, terrain, and a distant launch landmark are visible.
- Mouse/trackpad zoom, middle/space drag, toolbar controls, `Home`, `F`, and place centering work within bounded camera state.
- All places remain keyboard-selectable through the DOM mirror and stay synchronized with canvas selection.
- Reduced-motion mode stops ambient movement and makes camera transitions immediate.
- Browser diagnostics demonstrate the scene reaching the target frame rate on the local test environment.

## Results

- Added Phaser 3.90 as a dedicated, lazily loaded world-rendering chunk and established a typed bridge with bounded command queuing and explicit unsubscribe semantics.
- Added a fixture-derived scene definition without leaking the application projection or services into Phaser. Scene colors are parsed from inherited generated CSS custom properties.
- Rendered the authored 48 × 30 map with six distinct building silhouettes, lit windows, eight waypoint routes, terrain, harbor water, deterministic stars, clouds, wind streaks, a moon, and the distant launch landmark.
- Added deterministic canvas fitting that chooses the largest whole-number scale. The reviewed targets render at 18× (864 × 540 backing canvas at 1440 × 900) and 15× (720 × 450 at 1280 × 800).
- Added bounded integer-step wheel/toolbar zoom, middle- or Space-drag pan, place centering, Home reset, and `F` follow behavior. Reduced motion pauses ambient tweens and makes camera centering immediate.
- Kept six labeled place buttons in a coordinate-aligned DOM mirror. Keyboard selection updates Phaser highlighting, and Phaser hit-zone selection updates React's selected place.
- Added minimal fixture-agent camera targets for `F`; full character art and state synchronization remain intentionally assigned to P1-003.
- Hardened async lifecycle handling for React StrictMode with abortable lazy mounts and guaranteed bridge disconnect on scene shutdown or destruction.
- Added live diagnostics for renderer readiness, integer pixel scale, and FPS. The isolated headless browser gate sustained more than 30 FPS (typically mid-40s to high-50s during local validation).

## Remaining risks

- The minified Phaser runtime chunk is approximately 1.2 MB (319 KB gzip). It is code-split and fetched only for the world route, but production caching and deeper engine trimming should be reviewed after the vertical slice.
- Canvas rasterization produced a maximum 26-pixel difference across repeated 1440 × 900 headless captures. The visual assertion allows at most 100 changed pixels (less than 0.01% of the frame), which preserves meaningful regression sensitivity while absorbing renderer-edge noise.
