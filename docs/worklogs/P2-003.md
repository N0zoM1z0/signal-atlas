# P2-003 Worklog: Scripted fixture driver

## Status

Complete.

## Goal

Complete evidence-producing missions in fixture mode without Codex or Pref while preserving the same contracts, audit boundaries, deterministic identities, and recovery semantics expected from later live adapters.

## Intended files

- `apps/orchestrator`: seeded mission-script resolution, scheduled work, failure injection, result events, retry execution, and fixture configuration API.
- `packages/simulation`: recoverable-turn validation and safe failed-mission restart semantics.
- `apps/web/src/world-shell`: projection-backed evidence, offline scenario control, recoverable failure presentation, and retry commands.
- `tests/e2e/mission-flow.spec.ts`: successful scripted results, injected failure recovery, and the complete three-agent offline evidence run.

## Boundaries

- The driver reads authored fixture scripts and never calls Codex, Pref, or the network.
- A fixture seed plus mission ID, script key, and attempt number produce stable call and turn identities.
- Fixture entities remain outside the projection until their corresponding domain events are appended.
- Scenario selection is an adapter configuration. UI behavior branches only on generic scenario and projection state, never on Helios-3 agent or evidence IDs.
- A no-result response is a completed, evidence-free turn. Timeout and invalid-result responses are failed, recoverable turns that record no unvalidated entities.
- Retry is an explicit `runtime.retry_turn` command and retains the failed attempt in history.

## Acceptance criteria

- Authored mission latency gates all results and scales with authoritative simulation speed.
- Success emits Pref audit, source, claim, signal, explicit knowledge, belief, evidence-linked dialogue, turn, and mission events in dependency-safe order.
- Success, no-result, timeout, and invalid-result scenarios are selectable through a validated configuration boundary.
- Timeout and invalid-result missions can be retried only when the failed turn belongs to the same agent and mission and is marked recoverable.
- Identical fixture, commands, scheduler advances, and timestamps produce byte-equivalent result events.
- Mira, Orin, and Kestrel can each finish their authored Helios-3 evidence mission offline.
- The signal rail shows only evidence present in the authoritative projection.

## Results

- Added a deterministic fixture mission driver that resolves the authored `agent:verb:place` script key, clones only referenced entities, derives stable SHA-256-backed audit identities, and degrades unsupported keys to a valid no-result outcome.
- Extended the runtime with speed-aware work scheduling. A tick that ends travel does not double-count elapsed time toward newly-started work.
- Implemented the ordered success chain: `pref.call.started`, `source.recorded`, `pref.call.completed`, `claim.created`, `signal.created`, source/claim/signal knowledge edges, `belief.updated`, `agent.dialogue.emitted`, `agent.turn.completed`, and `agent.mission.completed`.
- Derived deterministic agent belief movement from the midpoint of each signal's authored impact range while preserving a normalized outcome distribution and recording assumptions and evidence IDs.
- Implemented evidence-free no-result completion and retryable timeout/invalid-result failures. Invalid payloads never cross the event parser or enter the projection.
- Activated the existing retry command with strict agent, mission, failed-turn, recoverability, and mission-status checks. Restarting work clears stale completion metadata while the earlier failure remains replayable.
- Added GET/PUT fixture configuration endpoints and an accessible Offline result selector. Recoverable failed missions remain in the queue with their error and a Retry action.
- Replaced staged fixture signal cards with projection-derived cards, so the initial evidence rail is truthful and discoveries appear only after result events.
- Added browser coverage for all three scenarios that matter to the player journey: normal travel/result completion, injected timeout recovery, and the complete Mira/Orin/Kestrel offline evidence run.
- Reviewed the refreshed 1440 × 900 and 1280 × 800 captures. The authored world remains balanced and the empty evidence state now communicates that source-linked evidence has not arrived yet.
