# P0-004 Worklog: Pure simulation and replay core

## Status

Complete.

## Goal

Turn the event catalog into a deterministic, infrastructure-independent world projection before UI and runtime behavior depend on it.

## Intended files

- `packages/simulation/src/state.ts`: sequence-zero projection builder and projection types.
- `packages/simulation/src/reducer.ts`: exhaustive, immutable event fold.
- `packages/simulation/src/selectors.ts`: agents, places, signals, knowledge, and forecast queries.
- `packages/simulation/src/commands.ts`: pure schema and world-legality validation skeleton.
- `packages/simulation/src/hash.ts`: canonical serialization and deterministic SHA-256 projection hash.
- `packages/simulation/src/replay.ts`: replay helpers and sequence slicing.
- `packages/simulation/test`: reducer, replay, selector, command, and purity tests.

## Projection boundary

- Market, world manifest, and agents are sequence-zero bootstrap data because the starter fixture is a snapshot plus a partial initial event stream.
- Fixture source, claim, and signal arrays remain an authored content catalog; they do not enter the authoritative projection until matching events record them.
- Agent knowledge changes only through `agent.knowledge.acquired`. Visibility and share events do not silently grant knowledge.
- Every accepted non-duplicate event must match the expedition and next contiguous sequence.
- Event timestamps are data. The reducer never reads the clock, performs I/O, or calls infrastructure services.

## Acceptance criteria

- All forty-five event variants are handled exhaustively.
- Replaying the same bootstrap and events produces the same SHA-256 hash.
- Duplicate event IDs return the existing projection without double application.
- Unknown versions, types, expedition IDs, non-contiguous sequences, and illegal transitions fail with typed diagnostics.
- Selectors expose agents, authored places, recorded active signals, explicit knowledge distribution, and forecast history.
- Command validation reports schema, idempotency, expedition, reference, and basic state-legality issues without executing side effects.
- Repository typecheck, lint, tests, format, and build pass.

## Results

- Added a sequence-zero builder that validates market/manifest identity, agent locations, and the absence of dynamic mission/knowledge state. It preserves authored market, world, agent, and belief bootstrap data while starting the event cursor at zero.
- Added a pure immutable projection covering agents, missions, recorded sources/claims/signals, explicit knowledge edges, correlations, meetings, Professor interactions, beliefs, forecasts, market prices, scores, dialogue, Pref calls, agent turns, and applied-event summaries.
- Implemented one exhaustive reducer case for each of the forty-five contract event variants. A test compares the contract union to reducer case labels exactly, and TypeScript's `never` exhaustiveness check protects future additions.
- Enforced event version, expedition identity, contiguous sequence, duplicate-ID idempotency, entity identity, route direction, monotonic travel progress, mission ownership, source/signal grounding, co-located meetings, explicit sender knowledge, probability continuity, and basic lifecycle legality.
- Added pure selectors for agents, authored places, recorded active signals, per-agent knowledge distribution, agents who know an object, and forecast history/latest forecast.
- Added schema-first command validation for all ten command variants, including world references, state legality, destination capabilities, and an idempotency ledger that binds each key to both command ID and canonical content hash.
- Added canonical JSON serialization and browser-compatible SHA-256 hashing with pinned `@noble/hashes` 2.2.0. The Helios-3 replay regression hash is `sha256:4f203ba904ee58469a39127a4ab0b5199e58deda07c333d9f5ad7fd7dd5836b3`.
- Added exact-sequence replay helpers. The fixture's authored source/claim/signal catalog remains absent from sequence-zero and sequence-two projections because no corresponding recording events have occurred.
- Added a deterministic `endsAt` field to `agent.travel.started`, so event producers calculate timing while the reducer only consumes recorded timestamps and never reads the clock.
- Added seventeen simulation tests covering bootstrap boundaries, fixture replay, fixed hashing, duplicate IDs, unknown versions/types, wrong expeditions, sequence gaps, frozen-state immutability, information events, explicit knowledge, stale signals, mission travel, command validation, and core purity constraints.
