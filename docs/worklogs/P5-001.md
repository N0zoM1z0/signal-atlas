# P5-001 Worklog: Pref Gateway fixture and canonical boundary

## Status

Complete.

## Goal

Create the only boundary allowed to turn external or recorded Pref responses into canonical `SourceRecord` objects. The first implementation is deterministic and fixture-backed, but it must expose the same canonical invocation contract, normalization, policy, provenance, and audit behavior required by future live MCP adapters.

## Intended files

- `packages/pref-gateway/package.json` and TypeScript/Vitest configuration: package dependencies, build, and focused tests.
- `packages/pref-gateway/src/types.ts`: runtime-validated configuration, canonical capability requests, call context, raw response, normalized result, diagnostics, audit event, and error contracts.
- `packages/pref-gateway/src/hash.ts`: deterministic canonical JSON and SHA-256 hashing for arguments, raw responses, content, and stable IDs.
- `packages/pref-gateway/src/normalize.ts`: rights-aware conversion of untrusted raw records into canonical contract-validated sources.
- `packages/pref-gateway/src/fixture-gateway.ts`: deterministic fixture implementation with connection state, exact recorded-request matching, allow-list enforcement, budgets, deadlines/timeouts, response-size limits, normalization, and audit emission.
- `packages/pref-gateway/src/index.ts`: focused public exports.
- `packages/pref-gateway/test/*.test.ts`: contract, policy, provenance, failure, and redaction tests.

## Acceptance criteria

- Recorded fixture responses pass through `PrefGateway.invoke` and the same normalizer intended for live adapters.
- Every returned source passes `SourceRecordSchema` and includes server, transport, primitive, primitive name, arguments hash, response hash, content hash, call ID, and retrieval time.
- Capability names and inputs are validated; capabilities outside the configured read-only allow-list fail closed.
- Calls fail explicitly when disconnected, over mission budget, past deadline, timed out, larger than the configured byte ceiling, or missing a recorded fixture.
- Audit events contain identifiers, hashes, timing, counts, and safe error codes/messages only. They never contain raw request values, source payloads, credentials, authorization headers, or connection secrets.
- Rights constraints remove stored content for metadata/link-only sources and prevent full structured payload storage for excerpt-only sources.
- Hashing and IDs are deterministic across equivalent object key ordering.
- Fixture mode remains fully offline and deterministic; no live transport or real-market write capability is introduced.

## Dependencies and risks

- Depends on the P0 contracts and existing canonical `SourceRecordSchema`.
- P5-002 will discover the actual Pref deployment and map real tools/resources/prompts onto this boundary. P5-001 must not assume the example primitive names are real.
- A timeout wrapper alone cannot guarantee cancellation of an arbitrary future transport. P5-002 adapters must propagate abort to the selected MCP client/transport.
- Raw response fixtures must be original or rights-safe and small enough for the repository; live/cache retention is deferred to P5-003.

## Results

- Added runtime-validated configuration and canonical request schemas for `search_sources`, `read_source`, and `local_conditions`. Configuration requires a read-only policy, unique capability allow-list, bounded timeout, response size, mission call count, and explicit cache mode.
- Added a `PrefGateway` interface with connection health, capability discovery, typed convenience calls, a generic canonical invocation path, public diagnostics, and structured failure codes.
- Added deterministic canonical JSON and SHA-256 helpers. Equivalent object key order produces the same argument, response, content, and source-identity hashes; cyclic and non-JSON-like values fail rather than hashing ambiguously.
- Added rights-aware normalization into the authoritative contract `SourceRecordSchema`. Full, excerpt-only, metadata-only, link-only, unknown-rights, disabled-cache, and metadata-cache behavior are explicit and tested.
- Preserved publication, observation, and retrieval timestamps as distinct fields. External IDs without a URI receive a stable percent-encoded `pref://` URI rather than being discarded.
- Added complete Pref provenance to every normalized source: server, fixture transport, primitive kind/name, argument hash, response hash, call ID, and content hash.
- Added an exact recorded-request fixture implementation. Recorded fixtures and future live adapters share the same invocation/result contract and source normalizer; fixture mode performs no network access.
- Enforced disconnected state, canonical input validation, capability denial, exact fixture matching, per-mission call budgets, absolute deadlines, per-call timeout, cancellation, and canonical response byte ceilings.
- Added paired start/completed/failed audit events that retain only stable identifiers, hashes, counts, byte/timing metadata, and fixed safe errors. Raw queries, payloads, authorization-like source content, and credentials never enter audit events or diagnostics.
- Added two test suites with 11 contract and policy tests, including a secret-bearing request/response fixture that proves audit and diagnostics remain clean.
- No real-market action, write capability, live MCP dependency, or network requirement was added.

## Remaining risks

- Fixture response size is measured from canonical recorded JSON after the test fixture is already in local memory. The P5-002 transport adapter must cap raw wire/stdout bytes before parsing or buffering an MCP response.
- The generic timeout wrapper safely rejects a fixture call but does not define live transport cancellation. P5-002 must propagate `AbortSignal` into the actual MCP client and terminate a stuck STDIO child or HTTP request.
- Stable source IDs generated without an adapter-supplied `sourceId` include the content hash, so changed content becomes a new identity. P5-003 adds cache/version matching and explicit `supersedesSourceId` behavior for a selected live capability.
- The gateway currently keeps mission budget counters in process memory. Durable cross-restart budget accounting can be added with the orchestrator event store when live automatic refreshes exist.
