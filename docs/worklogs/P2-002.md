# P2-002 Worklog: Travel and arrival simulation

## Status

Complete.

## Goal

Make mission movement event-driven, recoverable from the current projection, visibly synchronized with the authored world, and controllable without changing causal event order.

## Intended files

- `packages/simulation`: deterministic route planner, skip command validation, and movement-safe reducer transitions.
- `apps/orchestrator`: real-time fixture scheduler and ordered travel event production.
- `packages/game-scene`: distance-based waypoint interpolation and typed projection updates.
- `apps/web/src/world-shell`: authoritative speed/pause controls, projection polling, travel status, skip controls, and durable skip preference.
- `tests/e2e/mission-flow.spec.ts`: refresh, pause, speed, skip, arrival, and work-phase coverage.

## Boundaries

- Simulation time changes when speed changes; route choice and travel event order do not.
- Phaser interpolates a received projection. It never advances authoritative progress or decides that an agent arrived.
- Skip travel is an explicit command. It still appends remaining progress/arrival events before `agent.work.started` so replay remains truthful.
- The skip-travel checkbox is view preference only; applying it invokes the same audited command as the visible Skip travel action.
- Fixture mission results are not emitted in this milestone. P2-003 begins once the agent reaches the work phase.

## Acceptance criteria

- The authored minimum-duration route is selected deterministically, including multiple legs and reverse traversal.
- Travel emits start, monotonic progress, intermediate/final arrival, and final work-start events.
- Refresh reconstructs the traveling agent from the latest projection and movement continues.
- Pause freezes both movement and event sequence; resume continues the same plan.
- 1×, 2×, and 4× alter elapsed time without altering travel event order.
- Explicit and preferred skip travel preserve arrival events and enter the correct mission phase.
- Phaser moves the walk sprite along waypoints, follows the final arrival hint, and switches to work animation.

## Results

- Added a deterministic minimum-duration route planner with stable route-ID tie-breaking, direction-aware legs, and reversed waypoints for bidirectional traversal. The Helios-3 weather route correctly selects Observatory → Lantern Square → Weather Tower.
- Extended the command boundary with `agent.skip_travel` and regenerated the canonical command schema. Validation fails closed unless the named agent is moving for the named active mission.
- Added orchestrator travel scheduling at ten monotonic progress checkpoints per leg. Global speed scales simulated elapsed time, pause produces no ticks, intermediate arrivals start the next authored leg, and final arrival emits `agent.work.started`.
- Preserved the original expedition start time on resume and cleared stale movement when an active mission is canceled.
- Added typed `agent.project` scene commands and renderer acknowledgements. Phaser calculates positions by traveled polyline distance, smoothly updates sprites/highlights without remounting the scene, reverses visual paths when needed, and maps travel/work projections to walk/work animation.
- Replaced local ribbon toggles with authoritative pause/resume and speed commands. Added an accessible DOM progress bar, current leg destination, explicit Skip travel button, persistent skip preference, and final-arrival camera hint.
- Added a bounded projection poll only for local synchronization; command responses still force an immediate snapshot refresh. Test-only fixture reset is opt-in and keeps serial browser scenarios isolated.
- Added a root `predev` dependency build so `pnpm dev` works from a clean checkout before the orchestrator imports workspace runtime artifacts.
- Reviewed a live 1440 × 900 capture at 40% first-leg progress. Mira is visibly on the route, the dock exposes destination/progress/skip, and the queue retains the authoritative traveling phase.

## Remaining risks

- Projection synchronization is short polling rather than the planned WebSocket event cursor. P6-002 must replace or supplement it with reconnect-from-sequence behavior.
- Scheduled tasks are process-local, so an orchestrator restart cannot yet reconstruct elapsed timers. Browser refresh is covered; restart recovery belongs to persistence/replay work.
- The scheduler currently emits fixed 10% progress checkpoints. This is deterministic and adequate for the vertical slice, but long routes may later benefit from a configurable event density.
