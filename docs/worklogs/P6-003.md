## Acceptance criteria

- A fresh first run identifies the five-step fixture journey in context, can be skipped immediately, and never blocks ordinary world controls.
- Mission travel, work, evidence arrival, and completion are presented as a short ordered sequence derived from authoritative state; reduced-motion collapses decorative timing without hiding information.
- Weather changes cross-fade over a bounded interval, state and observation time are available as text, and reduced-motion applies the final state immediately.
- Sound is off until a user gesture enables it, uses only locally synthesized/original cues, has a persistent mute control, and every cue has a visible equivalent.
- Loading and working states name the operation and do not show fake model-call percentages.
- Capture mode hides diagnostic/test chrome while preserving provenance and runtime-mode truth; screenshot/video output contains no third-party art or debug UI.
- The 1440 x 900 and 1280 x 800 reference screens have intentional reviewed baselines with no clipping, overlap, accidental page scrollbar, or blurry world scaling.
- A scripted 30-minute fixture session completes without uncaught browser errors, event-stream corruption, timer/listener growth, or loss of interaction.
- World selection and camera controls remain responsive while an agent is traveling or working, and the scene meets an explicit, repeatable frame-rate/interaction-latency budget.
- Fixture replay remains deterministic and no real-market write or external action is introduced.

## Dependencies

## Risks and controls

- **Presentation becoming authority:** React derives ephemeral cue IDs from committed projections/events; neither Phaser nor audio callbacks may issue domain mutations.
- **Timers accumulating during a long session:** cue/tutorial timers use one owner, abort/cleanup on replacement and unmount, and receive fake-timer or browser longevity coverage.
- **Autoplay and sensory overload:** audio requires an explicit user gesture, defaults muted, stays subtle, and duplicates every cue in visible text; reduced-motion removes nonessential transitions.
- **Capture mode hiding trust state:** only renderer diagnostics, development affordances, and onboarding prompts may be suppressed. Fixture/live mode, connection state, source provenance, and forecast semantics remain visible.
- **Performance tests becoming flaky:** use generous lower-percentile FPS and upper-bound input-latency budgets after scene readiness, plus deterministic fixture state and a separately reported browser environment.
- **Visual churn:** keep the established palette/layout, update only intentional baselines, and manually inspect both required desktop sizes before accepting screenshots.
