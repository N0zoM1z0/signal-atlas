# P0-001 Worklog: Bootstrap monorepo

## Status

Completed.

## Goal

Create the production pnpm workspace and shared tooling while keeping the first change limited to repository bootstrap.

## Intended files

- Root workspace configuration: `package.json`, `pnpm-workspace.yaml`, TypeScript, ESLint, and Prettier configuration.
- `apps/web`: minimal React/Vite application with a branded Signal Atlas loading shell.
- `apps/orchestrator`: minimal Fastify application with a typed `GET /api/health` response.
- Required package directories from `AGENTS.md`, each with strict TypeScript package boundaries.
- Repository setup documentation and this worklog.

## Constraints

- TypeScript strict mode.
- Local services bind to `127.0.0.1` by default.
- Do not implement Phaser world behavior, domain contracts, Codex, Pref, persistence, or market actions in this task.
- Preserve the fixture and design references without modification.
- Keep packages independently buildable and avoid circular dependencies.

## Acceptance criteria

- `pnpm install` succeeds.
- `pnpm dev` starts the web application and orchestrator together.
- The web application displays a branded loading shell.
- `GET /api/health` returns typed JSON.
- `pnpm typecheck`, `pnpm lint`, `pnpm test`, and `pnpm build` pass.

## Validation commands

```text
pnpm install
pnpm typecheck
pnpm lint
pnpm test
pnpm build
pnpm dev
curl --fail http://127.0.0.1:4317/api/health
curl --fail http://127.0.0.1:4173/
```

## Dependencies and risks

- Requires registry access for workspace dependencies.
- The development process needs clean multi-process shutdown.
- Later packages will replace bootstrap-only placeholder exports with scoped implementations.

## Results

- A 12-project pnpm workspace containing the two applications and all nine required packages.
- Strict shared TypeScript configuration and repository-wide ESLint/Prettier foundations.
- A React/Vite loading shell using the Signal Atlas identity and reduced-motion-safe animation.
- A Fastify orchestrator bound to localhost with a typed fixture-mode health response.
- Root development documentation and explicit local-proxy handling in `AGENTS.md`.
