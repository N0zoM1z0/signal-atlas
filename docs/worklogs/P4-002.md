# P4-002 Worklog: Local schema-constrained Codex execution

## Status

Complete.

## Goal

Run one real, bounded Signal Atlas agent turn through the installed local Codex CLI without granting it repository-write access or allowing model output to mutate the world directly. The transport must be observable, cancelable, repairable once, evidence-reference checked, and safely replaceable with deterministic fixture behavior only when the executable is absent.

- `packages/codex-runtime/src/process.ts`: direct child-process execution and process-group termination.
- `packages/codex-runtime/src/local-exec-driver.ts`: command construction, isolated runtime files, validation, repair, safe wait, sessions, and diagnostics.
- `packages/codex-runtime/src/jsonl.ts`: privacy-preserving Codex event parsing.
- `packages/codex-runtime/src/prompt.ts`: bounded stdin packet and constrained repair packet.
- `packages/codex-runtime/src/redaction.ts`: stdout/stderr and error redaction.

- `packages/contracts/scripts/codex-output-schema.ts`: strict Structured Outputs transport schema.
- `apps/orchestrator/src/local-fixture-codex-driver.ts`: local/fixture driver selection, safe fixture context, and canonical world-artifact materialization.
- `apps/orchestrator/src/expedition-runtime.ts`: asynchronous scheduler integration that leaves the world loop responsive.
- `apps/orchestrator/scripts/run-local-codex-smoke.ts`: repeatable real-Mira verification.
- `apps/web/src/world-shell/RuntimeDiagnosticsDialog.tsx`: exact safe command and fallback-state display.

## Boundaries

- Only `packages/codex-runtime/src/process.ts` imports `node:child_process`. The orchestrator, simulation, game scene, and web application depend only on `CodexDriver` and scheduler contracts.
- The process is spawned with an executable plus argument array and `shell: false`; no prompt, path, model, or user value is interpolated into a command string.
- The initial turn uses an isolated per-agent runtime directory, `--sandbox read-only`, no repository working directory, and a bounded final-output file. Resume retains the recorded session while reasserting read-only policy through CLI configuration.
- User/project Codex configuration and exec-policy rules are ignored for gameplay turns. Shell tools, multi-agent work, apps, remote-plugin discovery, and web search are disabled explicitly.
- Model-spawned commands inherit no environment variables. The outer CLI process receives only a small runtime/auth-location/proxy/locale allow-list; generic API keys, tokens, secrets, and credentials are not forwarded.
- The prompt enters through stdin and is never included in the diagnostic command. Source excerpts are wrapped as untrusted evidence and explicitly cannot alter role, permissions, tools, or output shape.
- JSONL is parsed for session ID, usage, event type, item type, and status only. Agent-message text, reasoning content, commands, tool arguments, and tool results are never copied into runtime diagnostics.
- Stdout and stderr have hard byte ceilings. Non-zero process errors are extracted only from stderr or JSONL error events, redacted, truncated, and attributed to the Codex boundary.
- The final output file is size limited, consumed once, and removed. The strict transport schema is retained locally with mode `0600`; turn output is validated again with the canonical Zod schema after nullable transport fields are normalized.
- Output identities, capabilities, outcome IDs, signal IDs, and every source ID are checked against the scheduled turn packet and current evidence packet before materialization.
- A rejected output receives exactly one repair prompt containing only validation errors and the rejected output. A second rejection becomes a schema-valid safe wait with no accepted sources, claims, signals, or belief change.
- Only an actually missing executable activates the deterministic scripted fallback. Authentication, service, schema, and other process failures remain visible failures and are not disguised as successful fixture work.
- Codex selects and explains evidence, but authored fixture source, claim, and signal objects remain canonical. The model cannot authoritatively rewrite source text or provenance.
- Local turns run through `CodexTurnScheduler`; process latency happens outside the synchronous simulation tick, so travel, UI polling, and other world interactions remain responsive.

## Runtime command shape

The exact last invocation is retained in diagnostics as a redacted executable and argument array. The initial shape is:

```text
codex exec --sandbox read-only -C <isolated-agent-workspace>
  --ignore-user-config --ignore-rules --strict-config --skip-git-repo-check
  --json
  --disable shell_tool --disable multi_agent --disable apps --disable remote_plugin
  -c approval_policy="never"
  -c sandbox_mode="read-only"
  -c web_search="disabled"
  -c shell_environment_policy.inherit="none"
  --output-schema <runtime>/agent-turn-output.codex.schema.json
  -o <runtime>/<turn>-attempt-1.json
  -
```

Resume uses `codex exec resume` with the same supported config/output arguments, the captured session ID as a discrete argument, and `-` for the repair or next-turn stdin packet. It does not use the unsupported `resume --sandbox` or `resume -C` flags; `sandbox_mode="read-only"` reasserts the policy.

## Acceptance criteria

- The CLI is invoked directly without a shell and receives the prompt on stdin.
- Every initial local turn uses JSONL, a strict output schema, an output file, read-only sandbox policy, disabled external tools, and a bounded isolated workspace.
- `thread.started` captures the session ID; the next turn or repair uses `codex exec resume <session-id>`.
- A valid local result clears JSON Schema, canonical Zod, identity, capability, evidence-reference, and world validation before any event is appended.
- Invalid JSON receives one repair. Invalid JSON or an unknown source ID after repair accepts no source, claim, signal, or belief update.
- Timeout aborts the driver, signals the detached process group, escalates from `SIGTERM` to `SIGKILL` after a short grace period, and creates an explicit timed-out runtime/world failure.
- The app selects a visible deterministic fallback when the executable cannot be found.
- Diagnostics show the exact redacted last command and never include prompt text, agent-message text, private reasoning, credentials, or auth-file contents.
- Mira can consume the authored weather bulletin through real local Codex and produce the canonical crosswind signal.

## Remaining risks

- A read-only Codex shell sandbox is a write/network boundary, not a complete host-secret isolation boundary. This implementation additionally disables shell/tools, ignores user configuration, strips the model command environment, and uses an isolated workspace; a production deployment should still run gameplay Codex in a dedicated OS user, container, or VM.
- The strict transport schema intentionally fixes probability objects to the vertical slice's `yes`/`no` outcomes. The canonical runtime contract remains general; a future multi-outcome market needs a generated market-specific transport schema.
- Per-agent session IDs currently live only for the driver process lifetime. P4-003 adds versioned profiles, compact knowledge packets, and durable restart-safe session storage.
- Real model latency, availability, and account limits are inherently nondeterministic. Fixture mode remains the required default for tests and offline demonstrations.
- This milestone supplies authored fixture evidence to Codex. P5 routes live information through the audited Pref Gateway before it can enter the same packet.
