# P6-002 Worklog: Accessibility and resilience audit

## Status

In progress.

## Goal

Make the complete offline expedition keyboard-operable and semantically inspectable, preserve readable core state at 200% zoom and in forced-colors/reduced-motion environments, and ensure temporary Codex, Pref, event-stream, or schema failures remain boundary-specific without corrupting the authoritative projection.

## Intended files

- `apps/orchestrator/src/expedition-runtime.ts`, `app.ts`, and focused tests: authoritative event subscriptions, cursor-based WebSocket catch-up, malformed-cursor rejection, bounded stream envelopes, and reset-safe lifecycle behavior.
- `apps/web/src/world-shell/event-stream-client.ts`, `WorldShell.tsx`, and tests: reconnect with exponential backoff from the last applied sequence, event validation before cursor advancement, authoritative snapshot catch-up, and visible connection state.
- Runtime diagnostics and agent/world status components: distinguish local Codex unavailability with scripted fallback, Pref unavailability, orchestrator loss, and invalid agent-output/schema boundaries.
- World/Archive/Professor/Forecast/Replay components and styles: focus return, focus order, keyboard list navigation, semantic canvas mirrors, 200% zoom reflow, reduced motion, and forced-colors/high-contrast support.
- Playwright configuration and accessibility specs: keyboard-only required journey, reconnect continuity, failure-state integrity, 200% zoom, forced colors, canvas mirror parity, and automated accessibility scans.
- `README.md`, architecture/accessibility documentation, and this worklog: connection protocol, recovery semantics, accessibility controls, verification, and remaining risks.

## Acceptance criteria

- A keyboard-only user can select an agent, dispatch and confirm a mission, inspect the resulting source, enter Archive/Professor/Forecast/Replay, and return focus without pointer input.
- Every essential canvas place, agent, route/movement state, and available world action has a semantic DOM representation; no result or control exists only in Phaser.
- Core text and controls remain readable and reachable at browser 200% zoom without page-level horizontal overflow.
- Reduced motion removes nonessential scene motion; forced-colors mode preserves boundaries, focus, state names, and control affordances without relying on color.
- A temporary WebSocket loss leaves the last valid projection visible, reconnects using the last validated sequence, catches up without duplicates, and never advances the cursor for a malformed envelope.
- Local Codex unavailable, Pref unavailable, invalid agent output, invalid stream schema, and orchestrator unavailable each name the failed boundary and retain the last valid world state.
- Automated accessibility checks report no serious or critical violations on the World, Archive, Professor, Forecast, and Replay states.

## Risks and controls

- **Stream duplication or gaps:** Every connection declares `after=<lastValidatedSequence>`; envelopes are strictly parsed, contiguous event sequences are required, and a snapshot is fetched before accepting a gap.
- **Corrupt client projection:** Stream events never mutate the client projection directly. They trigger an authoritative snapshot catch-up only after envelope validation; the prior projection remains rendered on failure.
- **Reconnect storms:** One client owns one socket, uses bounded exponential backoff with jitter disabled in deterministic tests, and cancels timers on unmount.
- **Test-only fault leakage:** Fault injection remains behind `SIGNAL_ATLAS_E2E=1`, uses explicit testing routes/query values, and cannot alter production behavior.
- **Fallback ambiguity:** A missing local Codex executable is labeled as a scripted fixture fallback, not as a successful local runtime and not as a total application outage.
- **Automated-audit false confidence:** Axe results supplement, rather than replace, keyboard, zoom, forced-colors, screen-reader semantics, and manual focus review.
