# P6-001 Worklog: Resolved expedition replay and case file

## Status

Complete.

## Goal

Turn the Helios-3 expedition into a deterministic, inspectable case file: resolve the fictional market from its authored fixture, score eligible forecasts, replay the complete event stream at any recorded sequence, jump to evidence turning points, and export a provenance-separated record.

## Intended files

- `apps/orchestrator/src/expedition-runtime.ts` and tests: fixture-owned resolution, forecast scoring, exact-sequence replay, final hash parity, and idempotent resolution behavior.
- `apps/orchestrator/src/app.ts` and HTTP tests: read-only replay/case-file endpoints and a bounded fixture-resolution action that accepts no caller-supplied outcome.
- `packages/archive/src/index.ts` and tests: deterministic turning-point derivation and a public case-file serializer with explicit source, claim, signal, and forecast-rationale sections.
- `apps/web/src/world-shell/runtime-client.ts`, `ReplayWorkspace.tsx`, `WorldShell.tsx`, and styles: resolution controls, forecast timeline, turning-point navigation, native sequence scrubber, selected-sequence projection, and JSON export.
- Playwright journeys and visual baselines: complete the resolved-case workflow and guard its reference presentation.

## Acceptance criteria

- The authored `resolutionFixture` is the only source of the resolved outcome, timestamp, and note; the client cannot choose or forge them.
- Resolution appends `market.resolved`, one `score.calculated` event for every eligible forecast, and `expedition.resolved` in deterministic order.
- Brier scores and per-outcome components are covered by runtime and HTTP integration tests.
- Replaying the full event stream from sequence zero produces exactly the authoritative final projection hash.
- Every replay request selects either sequence zero or an event-backed sequence and returns a projection that contains no later state.
- Source and signal introduction markers expose their authoritative event sequence, and the UI can jump directly to it.
- The sequence scrubber is keyboard-operable and clearly reports the selected event, market state, evidence counts, forecast history, and score state.
- The exported public case file has distinct source, claim, signal, and forecast-rationale collections, includes its final projection hash and event history, and excludes private forecast memos.
- Fixture mode remains offline, deterministic, and free of real trading behavior.

## Risks and controls

- **Outcome forgery:** Resolution is a dedicated server operation with an empty body; values are copied only from the parsed expedition fixture.
- **Non-deterministic replay:** Replays use the sequence-zero fixture bootstrap plus the authoritative in-memory event log and compare canonical projection hashes.
- **Incomplete score identity:** Score events reference the reducer's forecast projection event ID, while the exported forecast record separately retains the user-facing commit ID.
- **Private-note leakage:** The case-file serializer constructs an allow-listed forecast-rationale shape and never serializes `privateMemo`.
- **False historical state:** Turning points derive from domain events, never entity timestamps or browser-local archive preferences.
- **Resolution races:** Repeated resolution calls return the existing result without appending duplicate events; resolution is rejected while agent work is still active.

## Results

- Added a fixture-owned resolution transaction. It accepts no player-selected outcome, refuses active or queued mission work, emits `market.resolved`, one score for each explicitly eligible forecast, and `expedition.resolved`, and returns the same batch without mutation on repeat calls.
- Added exact-sequence replay from the sequence-zero fixture bootstrap. The runtime rejects gaps and out-of-range targets and raises an invariant failure if the latest replay hash differs from the authoritative projection hash.
- Stabilized Brier decimals to twelve places so harmless binary floating-point tails do not leak into events, exports, UI labels, or canonical hashes.
- Added deterministic archive turning points for source versions, signals, forecasts, resolution, and scores. Source introduction navigation now opens the full replay at the event-backed sequence.
- Added a public case-file serializer with separate source, claim, signal, forecast-rationale, score, turning-point, and sanitized event sections. Forecast private memos are excluded from both the rationale records and copied event payloads.
- Added replay, case-file, and sealed fixture-resolution HTTP endpoints with explicit 400/409 boundaries.
- Added a three-column replay workspace with fixture resolution, a keyboard-native sequence range, previous/latest/next controls, event landmarks, selected world metrics, market state, forecast/score timeline, projection integrity, and a real JSON download.
- Updated the resolved market ribbon to display `Resolved NO`, close the forecast action, and disable ended simulation controls.
- Documented the implemented replay API, sealed resolution boundary, deterministic projection verification, and public-export privacy policy.

## Remaining risks

- Authoritative state and the event log are still process-local. Durable persistence, reconnect-from-last-sequence, and outage recovery are assigned to P6-002.
- `resolve-fixture` is intentionally specific to the offline vertical slice. A non-fixture product would require a separately authenticated resolution-source adapter and policy; accepting arbitrary browser outcomes must remain prohibited.
- The export currently includes the complete sanitized event stream in memory. Pagination or streaming may be needed for expeditions substantially larger than this bounded fixture.
- No real-money trading, order placement, or market execution path was added.
