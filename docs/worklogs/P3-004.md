# P3-004 Worklog: Professor Vale's evidence-bound study

## Status

Complete.

## Goal

Provide bounded evidence synthesis at Scholar's Hill: the player chooses exactly what Professor Vale may use, chooses a reasoning mode, receives a structured answer with visible assumptions and limitations, and can hand a suggested investigation back to the mission queue.

## Intended files

- `apps/orchestrator/src/fixture-professor-driver.ts`: deterministic seven-mode, evidence-bounded scripted response driver.
- `apps/orchestrator/src/expedition-runtime.ts`: Professor query, correlation assessment, and response event planning.
- `packages/simulation/src/commands.ts`: query identity and duplicate-selection validation.
- `packages/simulation/src/reducer.ts`: fail-closed query/response evidence boundary enforcement.
- `apps/web/src/world-shell/ProfessorWorkspace.tsx`: Scholar's Hill scene, evidence tray, modes, question form, response, and mission handoff.
- `apps/web/src/world-shell/WorldShell.tsx`: Professor workspace routing, authoritative query submission, and structured mission preparation.
- `tests/e2e/professor-workspace.spec.ts`: required correlation journey, insufficiency, keyboard modes, handoff, and visual regression.

## Boundaries

- A Professor response may cite only source and signal IDs explicitly present in its originating query. The reducer rejects any out-of-bound evidence even if a future driver attempts to emit it.
- The scripted driver reads discovered entities from the authoritative projection. The fixture supplies authored response prose for the canonical correlation case, not hidden evidence access.
- Distinct signal IDs are never treated as proof of independence. A successful correlation check records a `possibly_correlated` relationship with explicit reasons.
- Insufficient selections return a valid structured response beginning with “Insufficient evidence”; the driver does not silently add a second record.
- Professor responses synthesize evidence but do not commit a forecast or dispatch a mission. Suggested missions return to the ordinary player-confirmed command desk.
- Evidence selection is local draft state until Ask Professor appends the query event.

## Acceptance criteria

- Scholar's Hill opens from the world toolbar or the `P` keyboard shortcut.
- The evidence tray exposes discovered sources and signals with explicit selected counts and no hidden defaults beyond the latest shared meeting signals or case file.
- Explain, Challenge, Compare, Base rate, Missing evidence, Correlation check, and Forecast impact modes are present and left/right arrow navigable.
- The required weather-plus-history correlation check returns the authored bounded answer and records a correlation assessment.
- Responses render Answer, Evidence used, Assumptions, Limitations, Next best question, and Suggested mission sections when present.
- Every cited item is a subset of the submitted selection; unselected evidence fails closed in the reducer.
- Selecting only one signal for correlation clearly states insufficiency and cites only that one signal.
- Suggested mission handoff opens a normal Kestrel mission draft for player confirmation.
- The conventional text panel remains selectable, scrollable, keyboard-operable, and protected by screenshot regression.

## Results

- Added a deterministic Professor driver covering all seven authored modes. Each mode produces structured answer, evidence, assumptions, limitations, and next-question fields; insufficient selections produce explicit gaps instead of invented support.
- Adapted the fixture's canonical correlation response to each new query ID while filtering authored citations against the current selection. Generic responses are composed only from selected entities in the projection.
- Implemented authoritative `professor.query.started` and `professor.response.created` handling. A valid two-signal correlation check also appends one `correlation.detected` record unless that exact signal set has already been assessed.
- Added command validation for duplicate query IDs and duplicate source/signal selections.
- Hardened the reducer to require response mode consistency and enforce that `evidenceUsed` and `selectedSignalIds` are subsets of the originating query. A malicious unselected-source response is covered by a fail-closed reducer test.
- Built the Scholar's Hill scene with a cozy study, chalkboard reasoning boundary, discovered evidence tray, seven keyboard tabs, editable bounded question, and a semantic response panel.
- The response panel separates evidence used from assumptions and limitations, exposes a reusable next question, and prepares the scripted suggested mission through the existing confirmation workflow rather than auto-dispatching it.
- Defaults prefer the latest completed meeting's explicitly shared signals, then case-file signal pins, then the first discovered pair. This makes the required journey direct without hiding what was selected.
- Added driver tests across every mode, insufficiency, authored-response filtering, runtime correlation events, and reducer evidence-boundary rejection.
- Added a browser journey that completes both evidence missions and the meeting, opens the study with `P`, tests arrow-key modes, verifies the two-record bounded answer, proves one-record insufficiency, observes correlation labels, and hands the suggested mission to Kestrel.
- Added and manually reviewed the 1440 × 900 Professor Study baseline. The chalkboard boundary, selected evidence, modes, answer, citations, assumptions, limitations, and correlation state remain readable alongside the expedition rails.

## Remaining risks

- Generic fixture-mode responses are intentionally concise deterministic templates. P4 replaces the driver behind a bounded runtime interface without weakening the query evidence boundary.
- `possibly_correlated` is a qualitative audit label, not a measured coefficient. The UI and response limitations state that covariance is unavailable.
- The suggested mission is prepared for Kestrel by policy but still requires player confirmation; no autonomous dispatch occurs in this milestone.
