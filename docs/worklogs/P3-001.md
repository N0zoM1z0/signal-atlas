# P3-001 Worklog: Signal rail and source inspector

## Status

Complete.

## Goal

Make each discovered signal immediately useful for judgment while keeping source fact, extracted claim, agent interpretation, knowledge ownership, and belief effects visibly distinct.

## Intended files

- `apps/web/src/world-shell/model.ts`: projection-to-presentation adapter for source, claim, knowledge, correlation, and belief relationships.
- `apps/web/src/world-shell/SignalRail.tsx`: truthful category tabs, compact state-rich cards, knowledge chips, and pin actions.
- `apps/web/src/world-shell/SourceInspector.tsx`: accessible conventional source and provenance reading panel.
- `apps/web/src/world-shell/evidence-preferences.ts`: non-authoritative local case-file organization.
- `tests/e2e/signal-inspector.spec.ts`: source reachability, provenance, pin/archive persistence, keyboard dismissal, and visual regression.

## Boundaries

- Domain evidence always comes from the authoritative projection. The UI never reconstructs sources from the fixture.
- Seen, pinned, and archived IDs are personal view preferences stored locally; they do not mutate signal truth status or append fake domain events.
- New means unseen and not locally archived. Pinned and All may still contain archived evidence so organization never destroys history.
- Disputed, stale, and correlated labels are derived from domain signal/correlation state. An authored correlation-group ID without a correlation record is labeled “Independence unreviewed,” not asserted as correlation.
- Inspecting a signal marks only the local read state. Source excerpts remain selectable semantic text outside Phaser.

## Acceptance criteria

- Every active signal card opens its underlying source in no more than two interactions.
- Cards expose textual direction, impact range, freshness, reliability, source count, discoverer, knowledge holders, and state warnings without relying on color.
- The source inspector shows exact claims, source excerpt, source class/version, publication/observation/retrieval times, location, rights, stable identifier, hashes, and Pref provenance.
- New, Pinned, Disputed, and All tabs report truthful counts and remain keyboard-operable.
- Pin, archive, restore, and seen state persist across a browser reload without changing the domain projection.
- Five active cards remain available in the scrollable rail.
- A reviewed inspector screenshot is protected by regression testing.

## Results

- Enriched the shell presentation model with source and claim objects, explicit agent knowledge holders and acquisition mode, linked belief updates, correlation records, correlation groups, status labels, reliability reasons, freshness windows, and signed probability-point ranges.
- Reworked the signal rail into real New/Pinned/Disputed/All views. Cards now use arrow plus text direction, source counts, signed impact, freshness, reliability, discoverer, explicit knowledge chips, domain status, correlation state, and persistent pin controls.
- Added a wide, scroll-safe source inspector using the shared accessible dialog primitive. It separates Claims and assessment from Source layer, exposes complete provenance and timestamps, renders excerpts as selectable text, explains unreviewed independence, and reports linked belief updates.
- Added resilient local evidence preferences with deduplicated IDs and safe storage failure behavior. Archive removes a card from New while retaining it in All; restore returns it to New; pinning creates a durable case-file selection.
- Added model tests for stale, disputed, possibly-correlated, source, claim, and knowledge projections plus a five-card active set.
- Added a complete browser journey proving one-click source reachability, expected provenance fields, pin/archive/restore behavior, Escape dismissal, and persistence after reload.
- Added and manually reviewed the 1440 × 900 source-inspector baseline. Evidence hierarchy, selectable excerpt, knowledge ownership, independence caveat, hashes, and actions remain readable with the world visibly paused behind the modal.
