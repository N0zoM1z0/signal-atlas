# P1-001 Worklog: Main application frame

## Status

Complete.

## Goal

Translate the approved world-view composition into a production React shell with a dominant world stage, honest runtime status, responsive panel behavior, and keyboard-first navigation.

## Intended files

- `apps/web/src/world-shell`: fixture-backed shell model and five frame regions.
- `apps/web/src/App.tsx`: production world route while preserving `/components`.
- `apps/web/src/styles.css`: token-driven desktop grid, world-host treatment, panel collapse, and drawer breakpoints.
- `apps/web/src/App.test.tsx`: semantic shell assertions.
- `tests/e2e/world-shell.spec.ts`: interaction, overflow, responsive, status, and screenshot coverage.
- `tests/visual`: reviewed 1440 × 900 and 1280 × 800 baselines.

## Boundaries

- P1-001 owns frame composition and lightweight shell interactions, not authoritative mission execution.
- Market, agents, places, and latest forecast come from the P0 fixture projection.
- Signal cards are explicitly labeled staged fixture-preview content until P2 events record them into the projection.
- The central host is semantic React fallback content; P1-002 will mount Phaser behind the same toolbar and accessible place controls.
- All palette use comes from generated design tokens. No external assets or network calls are required.

## Acceptance criteria

- Five-part frame has no accidental page scrolling at 1440 × 900 or 1280 × 800.
- Agent and signal panels collapse independently; below the desktop breakpoint they become mutually exclusive drawers.
- Loading, Pref-disconnected, and fixture states are visible and screen-reader legible.
- Tab order follows ribbon → agents → world → signals → command tray, with `/` focusing the command field and number keys selecting agents.
- Screenshots are reviewed against the supplied world-view reference.
- Repository typecheck, lint, tests, format, build, and relevant Playwright tests pass.

## Results

- Replaced the loading-only route with a fixture-backed five-region application frame while preserving the `/components` foundation route.
- Derived market, agent, place, route, and staged-signal presentation from the validated P0 fixture and deterministic projection. Preview signals are visibly marked non-authoritative until mission events discover them.
- Added independent desktop panel collapse, mutually exclusive narrow-screen drawers, command-tray expansion, runtime state badges, URL-addressable loading/disconnected demonstrations, and an accessible live region.
- Added keyboard shortcuts for command focus (`/`), agent selection (`1`–`3`), pause (`Space`), speed (`[`/`]`), and drawer dismissal (`Escape`). Signal tabs implement roving focus with Left/Right arrows.
- Added functional Playwright coverage for overflow, responsive drawers, panel collapse, shortcuts, and runtime states. Added reviewed screenshot baselines at 1440 × 900 and 1280 × 800.
- Visual review confirmed the intended dark editorial pixel-diorama hierarchy: the world remains dominant, agent and evidence panels are dense but legible, the runtime badge is explicit, and both target desktop sizes preserve the command desk without body scrolling.
