# P3-005 Worklog: Deliberate forecast commits and scoring

## Status

Complete.

## Goal

Make a player forecast revision a deliberate, evidence-linked event rather than an unlabeled number change. The Observatory desk must keep the public market, team prior, and player path distinct; record a valid binary distribution; preserve private and public rationale boundaries; and prepare the resolved fixture for proper scoring.

## Intended files

- `packages/contracts/src/agents.ts` and `packages/contracts/src/events.ts`: non-empty public-note and durable commit-identity contracts.
- `packages/simulation/src/commands.ts`: authoritative forecast precondition and reference validation.
- `packages/simulation/src/reducer.ts` and `state.ts`: fail-closed forecast history projection.
- `packages/simulation/src/scoring.ts`: deterministic multiclass Brier scoring.
- `apps/orchestrator/src/expedition-runtime.ts`: accepted commit-to-event planning.
- `apps/web/src/world-shell/ForecastWorkspace.tsx`: accessible Observatory commit dialog and event history.
- `apps/web/src/world-shell/MarketRibbon.tsx`, `WorldShell.tsx`, and `WorldStageHost.tsx`: dynamic comparison values, opening controls, keyboard route, and authoritative submission.
- `tests/e2e/forecast-workspace.spec.ts`: required revision journey and visual regression.

## Boundaries

- The UI submits only `forecast.commit`; it never mutates the projection or forecast history directly.
- A revision must change the latest recorded distribution and link at least one discovered signal. An evidence-free commit is legal only as an explicit hold with unchanged probabilities.
- The command validator checks the previous distribution against the latest committed event, so two stale editors cannot silently overwrite one another.
- YES and NO are stored as one probability distribution whose keys exactly match the market outcomes and whose values sum to one. The UI derives NO as the complement of YES.
- Optional uncertainty ranges must include the committed point estimate for every outcome. The binary UI derives the complementary NO range from the entered YES interval.
- Player and agent commit actors must match the command actor. Initial system-authored team priors remain supported.
- Commit identity is carried into the event and projection independently of event identity, allowing duplicate detection and later export.
- Public notes are required and limited to 280 characters. Private memos remain visibly labeled local/private data in history.
- Brier scoring is calibration feedback only. It sums squared outcome errors from 0 (perfect) to 2 (maximum error) and has no monetary effect.

## Acceptance criteria

- The Observatory opens from both a visible `Commit Forecast` control and the `C` keyboard shortcut.
- A synchronized range dial and numeric input set YES from 0–100 while NO visibly complements it to 100%.
- Public market, latest team forecast, and prior player forecast remain separate values before and after a player commit.
- Optional uncertainty fields validate that the point estimate remains inside the range.
- Selected evidence appears as removable and reorderable signal chips; discovered signals can be added explicitly.
- A revision cannot be submitted without evidence, while a same-value hold may state that no new evidence was used.
- Public note, optional private memo, scoring explanation, and a consequential `Commit Forecast · N%` action are visible.
- The runtime appends one `forecast.committed` event carrying commit identity, previous/new distributions, evidence, uncertainty, and rationale.
- Forecast history is rendered from projection events and updates immediately after a successful commit.
- The authored resolved outcome can be scored with a tested Brier implementation.
- The interface contains no buy, sell, or bet action and has a reviewed 1440 × 900 screenshot baseline.

## Results

- Implemented forecast event planning in the fixture runtime. Accepted player commits now append one parsed `forecast.committed` event and fold it through the same reducer as every other state change.
- Added durable `commitId` propagation while preserving compatibility with the fixture's initial team-prior event.
- Hardened command validation against closed/resolved markets, duplicate commit and evidence IDs, stale previous values, forged player/agent actors, changed holds, unchanged revisions, evidence-free revisions, invalid uncertainty containment, and missing signal references.
- Mirrored critical invariants in the reducer so malformed replay or future driver output fails closed even if command validation is bypassed.
- Added deterministic multiclass Brier scoring with per-outcome components. The authored fixture's 48% YES forecast against the resolved NO outcome scores `0.4608`.
- Built a polished centered Observatory dialog over the living world. It includes a synchronized 0–100 dial, explicit YES/NO sum, revision delta, public/team/player comparison, optional uncertainty band, and clear simulated-record framing.
- Added ordered evidence chips with accessible move/remove controls, a discovered-evidence chooser, required public note, optional private memo, Brier-rule explanation, inline validation, and success/failure status.
- Rendered the forecast path from authoritative projection entries, including actor, previous-to-new delta, evidence labels, public rationale, optional private details, and any future score.
- Kept the ribbon's team value distinct after a player commit, added a dynamic worldline, exposed a toolbar route, and added the non-conflicting `C` shortcut (`F` remains camera follow).
- Used a monotonic recorded timestamp for fixture forecast commits so the authored 2027 history does not visually jump backward to the host clock.
- Added schema artifacts, contract/command/reducer/runtime/scoring/model tests, and an end-to-end journey that verifies the saved snapshot down to both complementary uncertainty ranges.
- Added and manually reviewed the 1440 × 900 Forecast desk baseline. The world remains visible behind the modal, while probability, evidence, rationale, success state, and two-entry history remain legible.
